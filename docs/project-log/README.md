# 项目日志归档

这个目录保存按日期拆分的详细操作记录。根目录 `PROJECT_LOG.md` 只维护最新进度、当前状态和下一步任务。

## 使用方式

- 每天的详细操作、调试过程、命令输出和学习记录写入对应的 `YYYY-MM-DD.md`。
- 如果当天已经有文件,就在文件末尾追加新的二级标题。
- `PROJECT_LOG.md` 只更新当前状态、关键结论、下一步任务和指向详细日志的链接。
- 不要把长命令输出、完整报错堆栈或大段 benchmark 明细继续塞回 `PROJECT_LOG.md`。
